# PRD: FCFS Finance — 선착순 금융 상품 가입 시스템

## 목표
은행 고금리 특판처럼 **수량이 한정된 금융 상품**에, 짧은 시간에 수만 명이 동시에 몰려도 **정확히 한정 수량만, 공정한 선착순으로, 1인 1건** 가입시키는 백엔드 시스템. RDB를 보호하면서 응답 지연을 최소화한다.

## 사용자 / 페르소나
- **일반 고객**: 알림을 받고 오픈 시각에 신청 버튼을 연타하는 사용자. 빠른 피드백(접수/마감/중복)을 원함.
- **(범위 밖) 상품 관리자**: 상품·수량 등록. MVP에서는 시드 데이터/내부 API로 대체.

## 핵심 기능 (Functional Requirements)
1. **회원가입** — 이메일 + 비밀번호. 이메일 유니크. 비밀번호 정책(최소 8자) 검증.
2. **로그인** — 자격 검증 후 JWT(access) 발급. 만료/위조 토큰 거절.
3. **상품 조회** — 목록/단건. 잔여 수량, 금리, 신청 기간(open/close), 상태(예정/진행/마감) 노출. 고빈도 조회는 캐시로 처리.
4. **선착순 신청** — 인증 사용자만. 결과: `접수(202)` / `마감(409)` / `중복(409)` / `기간외(409)`. 접수는 비동기 확정.
5. **신청 상태 조회** — `NOT_APPLIED` / `PENDING(처리중)` / `CONFIRMED(당첨확정)` / `FAILED_SOLD_OUT`.

## 비즈니스 규칙 (Business Rules) — 위반 시 전부 정의된 에러로 거절
- **BR-1 1인 1신청**: 동일 회원은 동일 상품에 1건만. 동시 더블클릭/재시도에도 멱등.
- **BR-2 수량 엄수**: 당첨자 수 ≤ 상품 수량. **오버셀 절대 0.** 음수 재고 불가.
- **BR-3 선착순 공정성**: 재고를 확보한 순서대로 당첨. 도착 순서 = Redis 원자 연산 처리 순서.
- **BR-4 신청 기간**: `openAt ≤ now < closeAt` 일 때만 접수. 그 외 거절(예정/마감).
- **BR-5 인증 필수**: 비로그인 신청 불가.
- **BR-6 확정 정합성**: 접수된 당첨은 반드시 DB에 1건 영속화(중복 없이). 시스템 장애에도 유실/중복 금지.

## 비기능 요구 (Non-Functional Requirements)
| 항목 | 목표치 |
|---|---|
| 동시 신청 | 10,000 VU 스파이크에서 정상 동작 |
| 신청 API p95 지연 | < 200ms (목표), 측정·기록 |
| 오버셀 | **0건** (불변식) |
| 신청 처리 정확성 | 성공 수 == 수량, 중복 0 |
| DB 보호 | 실패/중복/마감 요청은 DB 미접근 |
| 가용성 | Redis 장애 시 fail-fast(503), 데이터 무결성 우선 |

## MVP 제외 사항
- 실제 결제/이체/정산, 신용평가, 본인인증(PASS)
- 관리자 웹 UI, 상품 CRUD 화면(시드/내부 API로 대체)
- 이메일/SMS 발송, 소셜 로그인, refresh token 로테이션(access만)
- 멀티 인스턴스 무중단 배포(설계상 수평 확장 가능함은 문서로만 기술)
